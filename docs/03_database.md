# Database Schema - Heladería Dibuluc (MVP)

## Tables

### `products`
- `id` (INTEGER, PRIMARY KEY, GENERATED BY DEFAULT AS IDENTITY)
- `name` (VARCHAR(255), NOT NULL)
- `description` (TEXT)
- `price` (DECIMAL(10, 2), NOT NULL)
- `stock` (INTEGER, NOT NULL)
- `image_url` (VARCHAR(255))

### `orders`
- `id` (INTEGER, PRIMARY KEY, GENERATED BY DEFAULT AS IDENTITY)
- `customer_name` (VARCHAR(255), NOT NULL)
- `customer_phone` (VARCHAR(20), NOT NULL)
- `total_amount` (DECIMAL(10, 2), NOT NULL)
- `order_date` (TIMESTAMP, NOT NULL)

### `order_items`
- `id` (INTEGER, PRIMARY KEY, GENERATED BY DEFAULT AS IDENTITY)
- `order_id` (INTEGER, NOT NULL, FOREIGN KEY references `orders`.`id`)
- `product_id` (INTEGER, NOT NULL, FOREIGN KEY references `products`.`id`)
- `quantity` (INTEGER, NOT NULL)
- `price_at_purchase` (DECIMAL(10, 2), NOT NULL) # Precio del producto al momento de la compra

## Relationships

- One-to-many entre `orders` y `order_items`
- One-to-many entre `products` y `order_items`

## Endpoints (MVP)

- `GET   /api/products` — listado de productos
- `PATCH /api/products/:id/stock` — descuenta stock al confirmar pedido
- `POST  /api/orders` — crea pedido + order_items en una sola operación
